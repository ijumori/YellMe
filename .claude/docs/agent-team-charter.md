# YellMe エージェントチーム憲章

## 1. ミッション
- 最短で安全にリリースする。
- 個人記録アプリ方針（SNS要素なし）を守る。
- 実装と同時にドキュメントを更新する。

## 2. 役割固定（Chief差配）

### Chief（司令塔）
- チケット選定、優先順位、担当割り当て
- ブロッカー解除、仕様判断の最終決定
- マージ可否の最終判断

### PM Agent
- 受け入れ条件（3-5項目）を定義
- チケット分解（半日〜1日単位）
- 進捗記録、週次計画の更新

### Arch Agent
- MVVM整合性、責務分離、設計レビュー
- `project.yml` / XcodeGen運用の整合チェック

### Core Dev Agent
- Models/Services/同期ロジック実装
- データ整合、エラー処理、回復性の担保

### UI Dev Agent
- SwiftUI画面、導線、空状態、異常状態の実装
- 操作フローの一貫性と視認性を担保

### QA/Test Agent
- テスト観点作成、回帰テスト実行
- バグ再現手順と再発防止ケース追加

### Safety Agent（Double Check）
- Firestore Rules、認可、機密情報漏えいの監査
- リスク判断（出荷可否への警告）

### Release Agent
- TestFlight準備、リリースチェックリスト運用
- リリースノートとロールバック手順を更新

## 3. 責任分界ルール
- 1チケット1オーナー。補助役は1名まで。
- 実装者とレビュー者は分離する。
- セキュリティ観点はSafety Agentが最終確認する。
- 合意なしで方針変更しない（Chief承認必須）。

## 4. エスカレーション
- 30分以上詰まったらChiefへ即報告。
- リリース阻害バグはP0として当日中に判断。
- 不確実な仕様は「暫定仕様」で先に進め、PMが記録。
